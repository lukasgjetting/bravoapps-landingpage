FAQ

How can it be profitable to offer unlimited app development work?

By servicing a lot of mobile apps, we can invest heavily into automated systems and procedures. This allows us to do the work faster and more efficient.

Additionally, a lot of apps run into similar problems. For example, when Apple introduced App Tracking Transparency, every single app owner had to go and update their app.

What if I don't have enough app development work to keep you occupied?

We know that needs change. Sometimes, the app is just as it should be. In those cases, you shouldn't have to keep spending money on it!

Therefore, you can pause your subscription at any time. Then you can resume it, when you have work that needs to be done.

How do you define a "small" feature?

A small feature can be seen as an add-on to an existing feature.It is often not a standalone thing.

As this can be hard to define, we have written up some examples below.

Small features
  • Allow users to edit their tweets after writing and submitting them.
  • When a user has used the app 4 times, ask them to rate it.
  • Show the number of friends a user has, on their profile.
Big features
  • Implement a "friend system" so people can send, receive, accept and decline friend requests.
  • Add an advanced algorithm to show the most interesting tweets first.
  • Allow users to create their own character design with different clothes and hair styles.

What if I want a big feature?

Luckily, we can still assist. However, it will be as a separate contract outside the subscription. This also means that a big feature can be implemented in parallel with the rest of the maintenance work.
